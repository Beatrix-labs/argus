# Contributing to Argus

First off, thank you for considering contributing to Argus! It's people like you that make Argus a better tool for everyone.

## How Can I Contribute?

### Reporting Bugs
* Use the **Bug Report** template in GitHub Issues.
* Describe the steps to reproduce the issue.
* Attach a sample log line if possible (ensure sensitive data is redacted).

### Suggesting Enhancements
* Open an issue using the **Feature Request** template.
* Explain why this feature would be useful to the majority of users.

### Pull Requests (PR)
1. Fork the repository.
2. Create a new branch (`git checkout -b feature/amazing-feature`).
3. Commit your changes (`git commit -m 'feat: add some amazing feature'`).
4. Push to the branch (`git push origin feature/amazing-feature`).
5. Open a Pull Request.

## Coding Standards
* We follow the standard [Go Code Review Comments](https://github.com/golang/go/wiki/CodeReviewComments).
* Use `go fmt` before committing.
* Ensure your code is covered by comments for complex logic.

## Security
If you find a security vulnerability, please do NOT open a public issue. Instead, contact the maintainers directly at Beatrix-labs.
